# 方案可行性与边界

## 目标拆解

产品目标可以拆成四个模块：

1. 榜单采集：获得抖音或淘宝上某类商品的销量、价格、佣金、标题、主图、店铺等信息。
2. 素材采集：获得商品推广视频或图片，前提是素材来自自有账号、商家授权、达人授权或平台开放能力。
3. 素材理解：识别推广结构，包括痛点、镜头、字幕、卖点、价格锚点和行动号召。
4. 原创生成：基于结构生成新的脚本、分镜、海报文案或图片提示词。

## 平台现实

淘宝方向更容易先落地。淘宝开放平台提供淘宝客物料搜索接口，可以按关键词、类目、佣金、价格等条件获取推广物料。MVP 已放入 `taobao.tbk.dg.material.optional` 的适配入口。

抖音方向要区分公开视频数据、榜单数据和电商商品数据。抖音开放平台公开视频数据、榜单数据通常需要用户授权或权限申请；抖店商品和交易数据也需要商家/服务商授权。因此不能把“任意抓取抖音销量前十商品”作为默认能力，应改为“接入已授权数据源”。

## 风险

- 反爬风险：绕过登录、验证码、风控、频控抓取页面不可作为产品方案。
- 版权风险：竞品图片、视频、音乐、字幕、演员表演和剪辑都可能有版权或邻接权益。
- 平台规则风险：携带水印、品牌 logo、误导性导流、夸大功效可能导致限流、下架或封号。
- 广告法风险：绝对化用语、医疗功效、虚假销量背书都需要审核。

## 推荐产品形态

第一阶段做运营工作台：

- 用户选择官方数据源或上传授权 CSV。
- 系统筛选销量 Top 10。
- 系统分析素材结构，而不是复制素材本身。
- 系统生成原创广告方案和拍摄指令。
- 人工审核后进入拍摄、设计或投放。

第二阶段做自动化流水线：

- 增加定时任务拉取榜单。
- 增加素材库和版权台账。
- 增加视觉模型与文案模型。
- 增加品牌规则、违禁词和平台规则审核。
- 增加 A/B 测试和转化回流。

## 当前 MVP 取舍

当前版本先用样例数据和 CSV 导入打通闭环，并预留官方 API 适配器。这样可以快速验证产品经理关心的工作流，同时避免把不合规抓取作为核心能力。
