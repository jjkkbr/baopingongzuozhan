## 本版变化

- Electron 窗口内只允许工作台本地地址导航，阻止非工作台地址在桌面窗口内跳转。
- 外部链接只通过 `shell.openExternal()` 放行 `http:` 和 `https:`，阻断 `file:`、`javascript:`、`data:` 和自定义协议。
- 默认拒绝页面权限申请和权限检查，避免页面申请摄像头、麦克风、定位等桌面权限。
- 禁止附加 `webview`，降低嵌入外部页面带来的权限和导航风险。
- 下载只允许来自工作台本地地址，并兼容前端本地 `blob:` JSON 导出。
- 本地 HTTP 服务补充 `Content-Security-Policy`、`X-Content-Type-Options: nosniff`、`Referrer-Policy: no-referrer`、`X-Frame-Options: DENY` 和 `Permissions-Policy`。
- 静态资源服务使用路径相对关系校验，防止访问 `public/` 外部文件。
- UI 回归脚本新增安全响应头断言，后续改动会检查 CSP、`nosniff`、`no-referrer`、`DENY` 和权限策略。

## 已知问题

- 当前测试安装包未签名，Windows 仍可能提示“未知发布者”。
- AI 能力依赖用户自行配置可用模型和 Key；Key 类型、Base URL 和模型不匹配时会调用失败。
- Edge 插件识别率依赖真实授权页面 DOM，平台改版后需要补充授权样本并回归验证。
- 视觉风险分析是素材风险预检和人工复核辅助，不能替代法律、平台审核或版权审核结论。
- 历史销量趋势只展示真实导入字段，没有真实历史字段时不会生成模拟趋势。

## 合规边界

- 不实现绕过登录、验证码、风控、频控或反爬的能力。
- 不抓取未授权页面作为默认能力。
- 不下载、搬运、复刻竞品图片、视频、音乐、字幕、演员表演或剪辑。
- 不生成规避平台审核、规避版权或规避广告法的方案。
- 仅用于官方开放平台、商家后台导出、达人/商家授权数据、内部投放数据、用户有权访问和使用的页面内容，以及用户手动粘贴的授权文本、HTML 或 CSV。
