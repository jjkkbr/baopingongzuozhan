## 发布定位

这是 `v0.1.0` 之后的维护测试版，重点是升级 Electron 运行时和打包工具、补齐一键验证与 GitHub Actions，让测试版更适合继续迭代和对外试用。

## 本版变化

- Electron 从 `31.7.7` 升级到 `42.4.0`。
- electron-builder 从 `24.13.3` 升级到 `26.15.2`。
- `npm audit --audit-level=high` 已降为 0 漏洞。
- 新增 `npm run verify`、`npm run verify:syntax`、`npm run verify:edge` 和 `npm run verify:ui`。
- 新增 GitHub Actions 工作流 `.github/workflows/verify.yml`，push 和 PR 时自动运行依赖安装、基础验证和高危 audit。
- 适配 electron-builder 26 的 Windows 签名配置，证书主题和 SHA1 改为写入 `win.signtoolOptions`，未配置证书时不输出空签名字段。
- 继续保留 `npm.cmd run dist:signed`，正式证书仍通过环境变量传入。

## 已知问题

- 当前测试安装包未签名，Windows 可能提示“未知发布者”。
- AI 能力依赖用户自行配置可用模型和 Key；Key 类型、Base URL 和模型不匹配时会调用失败。
- Edge 插件识别率依赖真实平台页面 DOM，平台改版后需要补充授权样本并回归验证。
- 视觉风险分析是素材风险预检和人工复核辅助，不能替代法律、平台审核或版权审核结论。
- 历史销量趋势只展示真实导入字段，没有真实历史字段时不会生成模拟趋势。

## 合规边界

- 不实现绕过登录、验证码、风控、频控或反爬的能力。
- 不抓取未授权页面作为默认能力。
- 不下载、搬运、复刻竞品图片、视频、音乐、字幕、演员表演或剪辑。
- 不生成规避平台审核、规避版权或规避广告法的方案。
- 仅用于官方开放平台、商家后台导出、达人/商家授权数据、内部投放数据、用户有权访问和使用的页面内容，以及用户手动粘贴的授权文本、HTML 或 CSV。
